# Opciones para crear una App Móvil

## 1. React Native
### Pros
- JavaScript/TypeScript (mismo equipo puede trabajar en web y móvil)
- Gran comunidad
- Hot reload rápido
- Expo para desarrollo fácil sin CocoaPods

### Cons
- Rendimiento menor que nativo
- Librerías pueden tener problemas de compatibilidad

### Pasos
1. Instalar Node.js >= 18
2. `npx create-expo-app@latest mi-app`
3. `cd mi-app && npx expo start`
4. Desarrollar componentes con React
5. `eas build` para compilar APK/AAB

---

## 2. Flutter
### Pros
- Rendimiento casi nativo
- UI hermosa y consistente
- Documentación excelente
- Hot reload muy rápido

### Cons
- Dart (lenguaje específico)
- Curva de aprendizaje si no conoces Dart

### Pasos
1. Instalar Flutter SDK
2. `flutter doctor` para verificar
3. `flutter create mi_app`
4. `flutter run` para ver cambios
5. `flutter build apk` o `flutter build appbundle`

---

## 3. Capacitor (RECOMENDADO para tu proyecto)
### Pros
- Usa tu código web existente (Astro)
- Una sola base de código para web y móvil
- Acceso a plugins nativos (cámara, GPS, notificaciones)
- Compatible con Flask API existente

### Cons
- Rendimiento depende del webview
- Menos nativo que Flutter o React Native

### Pasos para tu proyecto (Flask + Astro)
```bash
# 1. En la carpeta E_3 (tu proyecto Astro)
npm create @capacitor/app@latest app-e3
# Seleccionar: Vanilla JS/TS, nombre: E3 App, ID: com.e3.app

# 2. Entrar a la carpeta
cd app-e3

# 3. Agregar Android
npx cap add android

# 4. (Opcional) Agregar iOS
npx cap add ios

# 5. Configurar la URL de tu API Flask
# Editar: capacitor.config.ts
# Agregar:
#   server: {
#     androidScheme: 'https',
#     hostname: 'tu-dominio-o-ip'
#   }

# 6. Hacer build de Astro
cd ..
npm run build

# 7. Sincronizar con Capacitor
npx cap sync

# 8. Abrir en Android Studio
npx cap open android

# 9. Compilar APK en Android Studio
# Build → Build Bundle(s) / APK(s) → Build APK(s)
```

### Integración con Flask API
```typescript
// En tu frontend Astro, ya usas fetch() para tu API Flask
// Eso funciona igual en Capacitor
const response = await fetch('https://tu-api-flask.com/api/socios');
```

### Plugins útiles para tu app
```bash
# Notificaciones push
npm install @capacitor/push-notifications

# GPS/Localización
npm install @capacitor/geolocation

# Cámara
npm install @capacitor/camera

# Almacenamiento local
npm install @capacitor/storage
```

## 4. NativeScript
### Pros
- TypeScript/JavaScript
- Acceso 100% a APIs nativas
- No WebView

### Cons
- Comunidad más pequeña
- UI diferente a web estándar

### Pasos
1. `npm install -g @nativescript/cli`
2. `ns create mi-app`
3. `ns run android` o `ns run ios`

---

## 5. PWA (Progressive Web App)
### Pros
- No requiere tienda de apps
- Se instala desde el navegador
- Sin approval de stores

### Cons
- No disponible en tiendas de apps
- Funcionalidades nativas limitadas

### Pasos
1. Crear web app responsive
2. Agregar manifest.json
3. Agregar service worker
4. HTTPS requerido
5. Probar con Lighthouse

---

## 6. Kotlin/Java para Android / Swift para iOS
### Pros
- Rendimiento máximo
- Acceso total a funcionalidades nativas

### Cons
- Dos bases de código separadas
- Curva de aprendizaje alta

### Pasos Android
1. Instalar Android Studio
2. New Project → Empty Activity
3. Desarrollar en Kotlin
4. Build → APK/AAB

### Pasos iOS
1. Instalar Xcode
2. New Project → App
3. Desarrollar en Swift
4. Build → Archive

---

## Recomendación

| Situación | Opción recomendada |
|-----------|-------------------|
| **Tu caso: Flask + Astro** | **Capacitor** |
| Ya tienes web app (React/Vue) | Capacitor |
| Nuevo proyecto, máximo rendimiento | Flutter |
| Nuevo proyecto, mismo equipo web | React Native + Expo |
| Solo Android | Kotlin |
| Prototipo rápido | PWA |
