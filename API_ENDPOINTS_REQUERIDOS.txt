ENDPOINTS API REQUERIDOS - PROYECTO E3
=====================================

Este documento lista los endpoints que la API debe implementar para que el frontend funcione correctamente.

---

API EXTERNA: https://e3-admin-api.onrender.com

---

ENDPOINTS EXISTENTES
--------------------

1. GET /api/socios
   Estado: ✅ Implementado
   Descripcion: Devuelve array de objetos para el carrusel (Features/Pillars)
   Respuesta:
   [
     {
       "id": 1,
       "title": "string",
       "description": "string",
       "image": "string (URL)",
       "contact": {
         "text": "string",
         "link": "string (URL)"
       }
     }
   ]

2. GET /api/componentes
   Estado: ✅ Implementado
   Descripcion: Devuelve array de todos los componentes configurables
   Respuesta:
   [
     {
       "nombre": "string",
       "titulo": "string",
       "subtitulo": "string",
       "contenido": "string",
       "link": "string",
       "extra_data": "string (JSON stringified)"
     }
   ]

3. GET /api/componentes/{nombre}
   Estado: ✅ Implementado
   Descripcion: Devuelve un componente especifico por nombre
   Respuesta:
   {
     "nombre": "string",
     "titulo": "string",
     "subtitulo": "string",
     "contenido": "string",
     "link": "string",
     "extra_data": "string (JSON)"
   }
   
   Nombres disponibles:
   - "Hero"
   - "AboutUs"
   - "MisionVision"
   - "CTA"
   - "Contact"
   - "Features"

  "valor4": "string",

---

NOTAS DE IMPLEMENTACION
-----------------------

1. Todos los endpoints deben manejar timeout de 5 segundos
2. Si la API falla, el frontend usa datos locales (fallback)
3. Los campos opcionales pueden ser null o no existir
4. Los slugs deben ser URL-friendly (kebab-case, sin espacios ni caracteres especiales)
5. Las imagenes deben ser URLs validas accesibles publicamente
